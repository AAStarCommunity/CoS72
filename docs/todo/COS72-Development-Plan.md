# COS72 任务广场开发计划

## 项目状态概览

### 当前阶段：Phase 1 MVP 核心循环验证
- **目标**：验证"任务换积分"核心商业逻辑
- **成功指标**：1个月内10个任务完成，用户接受新概念

### 技术栈
- **前端**：React + TypeScript + Vite
- **智能合约**：Solidity + OpenZeppelin
- **配置管理**：@aastar/shared-config
- **包管理**：pnpm

## Phase 1: MVP 核心功能 (优先级: 🔥 高)

### 📋 合约开发 (最高优先级)

#### 1.1 TaskContract 智能合约
- [ ] **状态管理**：实现任务状态枚举和结构体
  ```solidity
  enum TaskStatus { Open, InProgress, InReview, Completed, Cancelled, Disputed }
  ```
- [ ] **基础任务操作**：
  - [ ] `publishTask()` - 发布任务
  - [ ] `applyForTask()` - 申请任务  
  - [ ] `assignTask()` - 指派任务
  - [ ] `submitTask()` - 提交任务
  - [ ] `reviewTask()` - 审核任务
- [ ] **权限控制**：
  - [ ] 只有注册社区可发布任务
  - [ ] 只有任务发布者可审核任务
  - [ ] 只有被指派者可提交任务
- [ ] **事件定义**：为所有关键操作定义事件

#### 1.2 TaskFactory 工厂合约
- [ ] **合约部署**：`deployTaskContract()` 部署任务合约实例
- [ ] **地址管理**：`getTaskContract()` 获取社区任务合约地址
- [ ] **基础升级支持**：预留升级接口（完整实现在Phase 2）

#### 1.3 ProtocolTreasury 协议金库
- [ ] **收入计算**：1%协议费用自动计算和分离
- [ ] **收入分配**：70%给社区，30%给协议
- [ ] **金库管理**：xPNTs代币安全存储和分配

#### 1.4 集成现有合约
- [ ] **Registry集成**：验证社区注册状态和xPNTs地址
- [ ] **xPNTs集成**：自动积分发放和转账
- [ ] **Staking集成**：与现有stake系统关联

## 风险评估和缓解

### 🔴 高风险
- **合约安全**：通过多重审计和测试网验证缓解
- **用户体验**：持续的用户测试和反馈收集
- **Gas费用**：Layer2部署和优化策略

### 🟡 中风险
- **技术复杂度**：分阶段实施，优先核心功能
- **社区接受度**：早期用户教育和激励机制
- **监管风险**：合规性审查和法律咨询

### 🟢 低风险
- **技术可行性**：基于成熟的技术栈开发
- **市场需求**：已有明确的需求验证
- **竞争压力**：专注差异化功能和社区建设
